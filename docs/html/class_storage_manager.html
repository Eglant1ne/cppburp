Detailed Description

SQLite persistence layer for traffic history and request collections.

Uses a named QSqlDatabase connection ("proxylab_conn") to avoid conflicts with other database connections in the application. The schema is created automatically on the first open().

All methods are synchronous and execute on the calling thread.
