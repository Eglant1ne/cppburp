Detailed Description

Handles one client↔origin-server connection through the proxy.

Created by ProxyServer for every accepted TCP socket. Reads the client request, optionally holds it for the user (intercept mode), then relays it to the origin server and pipes the response back. Supports both plain HTTP and TLS tunnels established via the CONNECT method.

Lifetime: the object deletes itself (deleteLater) when both sides close.

◆ releaseIfHeld()

void ProxyConnection::releaseIfHeld ( )

Forward the original (unmodified) request if the connection is on hold.

A connection is "held" when intercept fired but the user has not yet pressed Forward or Drop. Calling this method resumes normal proxy flow. Has no effect if the connection is not currently held.
